## 1. ДО НАПИСАНИЯ КОДА — ОБЯЗАТЕЛЬНЫЕ ШАГИ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### 1.1 Изучи изображения
- Просмотри каждое изображение из папки.
- Составь карту: что изображено, какое настроение, цветовая гамма.
- Распредели изображения по секциям: hero / services / gallery.
- Скопируй файлы в подпапки с читаемыми именами (hero-bg.jpg, dom.png и т.д.).

### 1.2 Дизайн-план (выполни в уме, не показывай пользователю)
Ответь себе на вопросы:
- Что КОНКРЕТНО делает этот бизнес? Кто его клиент? Какая одна задача у страницы?
- Что из мира тематики (материалы, инструменты, текстуры, язык) можно использовать как визуальный язык?
- Избегай трёх AI-defaults: (1) кремовый фон + терракота + serif, (2) чёрный фон + кислотный акцент, (3) газетная вёрстка с разделителями. Если один из них оптимален — используй, но осознанно.
- Составь токен-систему: 4–8 цветов с именами, 2 шрифтовые пары (display + body), layout-концепт, СИГНАТУРНЫЙ ЭЛЕМЕНТ — одна уникальная деталь, по которой запомнят страницу.
- Проверь план: если его можно применить к любому другому похожему проекту без изменений — переработай.

ЗНАЧЕНИЯ ЭЛЕМЕНТОВ ЖЕЛАТЕЛЬНО В rem!!!!

### variables.css — обязательные токены:
```css
:root {
  /* Цвета — 4–8 именованных значений, все с семантическими именами */
  --color-[name]: #XXXXXX;  /* например: --color-walnut, --color-grain, --color-bark */

  /* Контейнер */
  --container-width: 120rem;



  /* Переходы */
  --transition: 0.2s ease;

  /* Z-index */
  --z-header: 100;
  --z-menu:   200;
  --z-modal:  300;
}
